# 区块链架构与共识机制

区块链（Blockchain）是一种分布式账本技术，通过密码学保证了数据的不可篡改性。

## 1. 核心数据结构
- **区块 (Block)**：包含交易数据、时间戳以及当前块的哈希值。
- **区块头 (Block Header)**：包含**前一区块哈希 (Previous Hash)**。这种引用关系构成了“链”。
- **梅克尔树 (Merkle Tree)**：一种通过计算哈希值组织的树状结构，用于高效验证区块内成千上万笔交易的完整性。

## 2. 核心密码学技术
- **哈希函数**：确保数据只要变动一个字节，整个区块链的哈希值就会失效。
- **非对称加密**：用户使用私钥给交易签名，全网使用公钥验证。
- **工作量证明 (PoW)**：通过消耗计算资源（挖矿）来竞争记账权。

## 3. 共识机制对比

| 机制 | 名称 | 优点 | 缺点 | 场景示例 |
| :--- | :--- | :--- | :--- | :--- |
| **PoW** | 工作量证明 | 极致安全、去中心化 | 极度耗能、延迟高 | 比特币 (BTC) |
| **PoS** | 权益证明 | 节能、高吞吐、利息收益 | 容易导致财富集中 | 以太坊 2.0 (ETH) |
| **DPoS** | 代理权益证明 | 速度极快 | 牺牲了部分去中心化 | EOS |

## 4. 智能合约 (Smart Contracts)
- **定义**：一段部署在区块链上的、自动执行的代码。
- **特性**：一旦部署，无法修改（除非代码预留了升级槽位），执行结果公开透明。
- **代表项目**：以太坊（全球第一台分布式计算机）。

## 5. 公有链 vs 私有链 vs 联盟链
- **公有链**：任何人可读可写可参与（如比特币）。
- **联盟链**：仅限预先设定的成员参与（如 Hyperledger Fabric，适用于金融清算）。
- **私有链**：单个组织内部使用，通常用于内部数据审计。

## 6. 区块链的“不可能三角”
无法同时实现：**去中心化 (Decentralization)**、**安全性 (Security)** 和 **可扩展性 (Scalability)**。
目前的扩容方案（如侧链、分片）本质上是在这三者之间进行权衡。
