# PaiSmart 系统优化与演进计划

本计划旨在通过引入前沿的检索增强技术 (RAG) 支持，将系统从“可用”提升至“生产级高性能”。

## 1. 核心检索与 RAG 优化 [✅ 全部完成]

### 1.1 意图治理与查询转换 (Query Transformation) [✅ 已完成]
- **实现**: 引入 LLM 预处理层实现指代消解（代词补全）、语义重写（口语转书面）及多路查询扩写（一次提问生成 3 个变体），并同步恢复 100% 严格匹配策略。
- **收益**: 彻底解决了追问时丢失上下文及口语表达导致的漏召回问题。

### 1.2 引入重排序 (Rerank) [✅ 已完成]
- **实现**: 集成 Rerank 模型对 Top-50 候选集进行精排。
- **收益**: 召回精度大幅提升。

### 1.3 融合算法升级 (RRF) [✅ 已完成]
- **实现**: 使用 **Reciprocal Rank Fusion (RRF)** 算法融合多路召回分布。
- **收益**: 消除手动权重的偏差。

### 1.4 并发与批量化优化 (Performance) [✅ 已完成]
- **实现**: 增加**批量向量化 (10/Batch)** 和 **并发 Worker 池 (5 Workers)**。
- **收益**: 向量化效率提升 400%+，API 成本大幅降低。

### 1.5 动态匹配阈值调优 (Dynamic Thresholding) [已由 1.1 方案替代]

## 2. 系统级功能增强

### 2.1 语义缓存 (Semantic Cache) [待开始]
- **场景**: 避免重复回答相似问题，降低 LLM Token 成本。
- **实现**: 使用 Redis 存储 `Query_Vector -> LLM_Response` 映射。

### 2.2 智能查询重写 (Query Rewriting) [✅ 已完成]
- **实现**: 已整合至 1.1 的查询转换流水线中。

### 2.3 文档解析升级 [进行中]
- **现状**: 已实现 **CodeSplitter (函数级感知识别)**。
- **优化**: 拟引入专门的 PDF 解析器 (如 PyMuPDF)。
