### 1.2 可优化点

#### 1.2.1 动态缓存策略

**当前问题:**
- 缓存 TTL 固定为 24 小时，不够灵活

**优化方案:**
```go
// 根据查询频率动态调整 TTL
func (s *SearchService) getDynamicTTL(query string) time.Duration {
    freq := s.getQueryFrequency(query)
    switch {
    case freq > 100:  // 高频查询
        return 7 * 24 * time.Hour
    case freq > 10:   // 中频查询
        return 24 * time.Hour
    default:          // 低频查询
        return 1 * time.Hour
    }
}
```

#### 1.2.2 批量向量化

**当前问题:**
- `processor.go:138` 逐个分块调用 embedding API，效率低

**优化方案:**
```go
// 批量向量化（每批 10 个）
const batchSize = 10
for i := 0; i < len(savedVectors); i += batchSize {
    end := i + batchSize
    if end > len(savedVectors) {
        end = len(savedVectors)
    }
    batch := savedVectors[i:end]
    texts := make([]string, len(batch))
    for j, dv := range batch {
        texts[j] = dv.TextContent
    }
    vectors, err := p.embeddingClient.CreateEmbeddingBatch(ctx, texts)
    // 批量索引到 ES
}
```

**预期收益:**
- API 调用次数减少 90%
- 处理时间缩短 60-70%

## 2. 文档处理流程优化

### 2.1 自适应分块策略

**当前问题:**
- 固定分块大小 1000 字符，不适合所有文档类型

**优化方案:**
```go
type ChunkStrategy interface {
    Split(text string) []string
}

// Markdown 语义分块
type MarkdownChunker struct {
    maxChunkSize int
}

func (m *MarkdownChunker) Split(text string) []string {
    // 按标题层级分块
    sections := parseMarkdownSections(text)
    chunks := []string{}
    for _, section := range sections {
        if len(section) > m.maxChunkSize {
            // 递归分割
            chunks = append(chunks, m.splitLargeSection(section)...)
        } else {
            chunks = append(chunks, section)
        }
    }
    return chunks
}

// 代码文件分块
type CodeChunker struct{}

func (c *CodeChunker) Split(text string) []string {
    // 按函数/类分块
    return parseCodeBlocks(text)
}
```

### 2.2 文档类型识别

**建议增加:**
```go
func (p *Processor) selectChunker(fileName string) ChunkStrategy {
    ext := filepath.Ext(fileName)
    switch ext {
    case ".md":
        return &MarkdownChunker{maxChunkSize: 1500}
    case ".go", ".java", ".py":
        return &CodeChunker{}
    default:
        return &DefaultChunker{chunkSize: 1000, overlap: 100}
    }
}
```

## 3. 并发与资源管理

### 3.1 并发向量化

**当前实现 (串行):**
```go
for i, docVector := range savedVectors {
    vector, err := p.embeddingClient.CreateEmbedding(ctx, docVector.TextContent)
    // ...
}
```

**优化方案 (并发):**
```go
const maxWorkers = 5
sem := make(chan struct{}, maxWorkers)
var wg sync.WaitGroup
errChan := make(chan error, len(savedVectors))

for _, docVector := range savedVectors {
    wg.Add(1)
    go func(dv *model.DocumentVector) {
        defer wg.Done()
        sem <- struct{}{}        // 获取信号量
        defer func() { <-sem }() // 释放信号量

        vector, err := p.embeddingClient.CreateEmbedding(ctx, dv.TextContent)
        if err != nil {
            errChan <- err
            return
        }

        esDoc := model.EsDocument{
            VectorID:    fmt.Sprintf("%s_%d", dv.FileMD5, dv.ChunkID),
            Vector:      vector,
            // ...
        }
        if err := es.IndexDocument(ctx, p.esCfg.IndexName, esDoc); err != nil {
            errChan <- err
        }
    }(docVector)
}

wg.Wait()
close(errChan)

// 收集错误
for err := range errChan {
    log.Errorf("并发处理失败: %v", err)
}
```

**预期收益:**
- 处理速度提升 3-5 倍

## 4. 错误处理与重试机制

### 4.1 死信队列 (DLQ)

**当前问题:**
- Kafka 消费失败后无重试机制

**优化方案:**
```go
type TaskRetryManager struct {
    maxRetries int
    dlqTopic   string
}

func (m *TaskRetryManager) HandleFailure(task tasks.FileProcessingTask, err error) {
    task.RetryCount++
    if task.RetryCount >= m.maxRetries {
        // 发送到死信队列
        m.sendToDLQ(task, err)
        log.Errorf("任务失败超过最大重试次数，已发送到DLQ: %+v", task)
        return
    }

    // 指数退避重试
    backoff := time.Duration(math.Pow(2, float64(task.RetryCount))) * time.Second
    time.Sleep(backoff)
    m.retryTask(task)
}
```

### 4.2 部分失败容错

**优化建议:**
```go
func (p *Processor) Process(ctx context.Context, task tasks.FileProcessingTask) error {
    // ...

    failedChunks := []int{}
    for i, docVector := range savedVectors {
        vector, err := p.embeddingClient.CreateEmbedding(ctx, docVector.TextContent)
        if err != nil {
            log.Warnf("分块 %d 向量化失败，跳过: %v", docVector.ChunkID, err)
            failedChunks = append(failedChunks, docVector.ChunkID)
            continue  // 不中断整体流程
        }
        // ...
    }

    if len(failedChunks) > 0 {
        log.Warnf("文件 %s 有 %d 个分块处理失败: %v",
            task.FileMD5, len(failedChunks), failedChunks)
    }

    return nil  // 部分成功也返回 nil
}
```

class WebSocketClient {
    constructor(url) {
        this.url = url;
        this.reconnectDelay = 1000;
        this.maxReconnectDelay = 30000;
        this.connect();
    }

    connect() {
        this.ws = new WebSocket(this.url);

        this.ws.onclose = () => {
            console.log('连接断开，尝试重连...');
            setTimeout(() => {
                this.reconnectDelay = Math.min(
                    this.reconnectDelay * 2,
                    this.maxReconnectDelay
                );
                this.connect();
            }, this.reconnectDelay);
        };
    }
}
